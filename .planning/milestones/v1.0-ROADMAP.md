# Milestone v1.0: MVP

**Status:** SHIPPED 2026-01-23
**Phases:** 1-7
**Total Plans:** 20

## Overview

Ontology Hub transforms a wiki admin's local schema changes into a GitHub PR through seven progressive phases: foundation infrastructure, GitHub indexing, entity browsing, modules and versioning, draft proposal system, validation engine, and PR integration. Each phase builds on the previous, with browsing features depending on indexed data, drafts comparing against indexed schemas, validation analyzing drafts, and PR creation consuming validated changes. The journey prioritizes security architecture early (capability URLs, token handling) since these cannot be retrofitted.

## Milestone Summary

**Key Decisions:**
- Fragment-based capability URLs reduce referrer leakage
- SHA-256 token hashing, never store plaintext
- Unified 404 for invalid/expired tokens (no oracle attacks)
- GITHUB_TOKEN optional at startup for better DX
- Zustand with immer middleware for draft editing state
- graphlib.TopologicalSorter for cycle detection
- Breaking changes as warnings not errors
- Git Data API for atomic multi-file commits

**Issues Resolved:**
- Rate limiting Retry-After header calculation
- TypeScript erasableSyntaxOnly compatibility
- Fragment-based capability URL redirect pattern
- Background task session management

**Issues Deferred:**
- Canonical vs local overlay toggle (v2)
- Release artifact indexing (v2)
- Mobile-optimized UI (v2)

**Technical Debt Incurred:**
None - all phases completed without shortcuts.
